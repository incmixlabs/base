# InfiniteTable

A high-performance, virtualized data table with built-in filtering, sorting, column resizing, and infinite scroll support. Built on **TanStack React Table** and **TanStack React Virtual**.

## Two APIs

| API | Use case |
|---|---|
| **`InfiniteTableWrapper`** | Drop-in single component — pass columns, data, and filter config. Handles layout internally. |
| **`InfiniteTable.*`** (compound) | Full control over layout — compose `Root`, `Toolbar`, `Layout`, `FilterPanel`, `Content`, and `Footer` yourself. |

---

## Wrapper (Quick Start)

The simplest way to get a filterable, sortable, virtualized table:

```tsx
import { InfiniteTableWrapper } from "@incmix/ui/table/infinite";

<InfiniteTableWrapper
  columns={columns}
  data={data}
  filterFields={filterFields}
  size="sm"
  footer={<span>500 rows</span>}
/>
```

### Shared Cell Renderers

The preferred reusable column schema is `TableColumnDef` from `@incmix/ui/table/basic`. Define portable columns with that
schema, then adapt them with `toTanStackTableColumnDefs` when rendering `InfiniteTableWrapper`, `InfiniteTable.*`, or
`TreeTable.*`.

```tsx
import { toTanStackTableColumnDefs, type TableColumnDef } from "@incmix/ui/table/basic";
import { InfiniteTableWrapper } from "@incmix/ui/table/infinite";

const sharedColumns: TableColumnDef<Row>[] = [
  {
    id: "owner",
    header: "Owner",
    accessorKey: "owner",
    width: 220,
    rowHeader: true,
    format: { renderer: { type: "avatar" } },
  },
  {
    id: "status",
    header: "Status",
    accessorKey: "status",
    format: { renderer: { type: "status" } },
  },
];

const columns = toTanStackTableColumnDefs(sharedColumns);

<InfiniteTableWrapper columns={columns} data={rows} />;
```

`InfiniteTableColumnDef` remains the low-level TanStack-compatible escape hatch for implementation-specific features such
as custom `cell` functions, custom filter functions, or table behavior that is not represented by the shared schema yet.

<Canvas of={WrapperStories.SharedCellRenderers} />

The same schema can carry editable metadata (`editable`, `editor`, `validateEdit`) when the table also receives an
`onCellEdit` handler. Use the low-level column type only when you need behavior outside that shared contract.

### Wrapper Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `columns` | `InfiniteTableColumnDef<TData>[]` | required | Low-level TanStack-compatible column definitions. Prefer `TableColumnDef` + `toTanStackTableColumnDefs` for reusable table schemas. |
| `data` | `TData[]` | required | Row data |
| `filterFields` | `FilterField[]` | — | Declarative filter config (enables sidebar) |
| `toolbar` | `boolean` | `true` | Show/hide toolbar |
| `toolbarActions` | `ReactNode` | — | Custom actions in toolbar right slot |
| `filterPanel` | `boolean` | auto | Override filter panel visibility |
| `defaultFilterPanelOpen` | `boolean` | `true` | Whether the filter panel starts open |
| `defaultColumnFilters` | `ColumnFiltersState` | — | Pre-applied column filters |
| `footer` | `ReactNode` | — | Footer content |
| `size` | `xs \| sm \| md \| lg` | `sm` | Table density |
| `variant` | `surface \| ghost` | `ghost` | Visual variant |
| `getRowId` | `(row) => string` | — | Row identity function |
| `hasNextPage` | `boolean` | — | Enable infinite scroll |
| `fetchNextPage` | `() => void` | — | Fetch next page callback |
| `isLoading` | `boolean` | — | Show loading state |

## Compound API (Full Control)

For custom layouts — e.g., toolbar on the side, custom filter placement, extra panels:

```tsx
import { InfiniteTable } from "@incmix/ui/table/infinite";

<InfiniteTable.Root columns={columns} data={data} filterFields={filterFields} size="sm">
  <InfiniteTable.Toolbar actions={<Button>Export</Button>} />
  <InfiniteTable.Layout>
    <InfiniteTable.FilterPanel />
    <InfiniteTable.Content />
  </InfiniteTable.Layout>
  <InfiniteTable.Footer>
    <span>Showing 500 rows</span>
  </InfiniteTable.Footer>
</InfiniteTable.Root>
```

### Sub-components

| Component | Description |
|---|---|
| `InfiniteTable.Root` | Context provider — takes columns, data, filterFields, size, variant |
| `InfiniteTable.Toolbar` | Top bar with filter toggle, row count, reset, and custom actions slot |
| `InfiniteTable.Layout` | Flexbox container for sidebar + content |
| `InfiniteTable.FilterPanel` | Accordion-based filter sidebar (checkbox, input, slider) |
| `InfiniteTable.Content` | Virtualized table with sticky header, sorting, column resizing |
| `InfiniteTable.Footer` | Bottom bar slot |

## Filter Fields

Filters are configured declaratively via the `filterFields` prop:

```ts
const filterFields = [
  {
    id: "method",          // must match a column accessor
    label: "Method",
    type: "checkbox",      // checkbox | input | slider
    defaultOpen: true,
    options: [
      { label: "GET", value: "GET" },
      { label: "POST", value: "POST" },
    ],
  },
  {
    id: "latency",
    label: "Latency (ms)",
    type: "slider",
    min: 0,
    max: 5000,
    step: 50,
  },
];
```

### Filter Types

| Type | Description | Extra props |
|---|---|---|
| `checkbox` | Multi-select checkboxes with optional counts | `options` |
| `input` | Free-text search | — |
| `slider` | Range slider | `min`, `max`, `step` |

## Features

- **Row virtualization** — renders only visible rows via `@tanstack/react-virtual`, handles 10k+ rows smoothly
- **CSS variable column sizing** — each column width is a CSS custom property (`--col-{id}-size`) for performant resizing
- **Column resizing** — drag column borders to resize, use the focused separator with ArrowLeft/ArrowRight (hold Shift for larger steps), double-click to reset
- **Sorting** — click column headers to sort (asc/desc/none cycle)
- **Filtering** — pluggable filter types with faceted value counts
- **Infinite scroll** — pass `hasNextPage` + `fetchNextPage` for server-side pagination
- **Sticky header** — header stays fixed during scroll
- **Composable** — use the wrapper for quick setup or compound components for full control
