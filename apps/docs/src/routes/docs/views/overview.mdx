<Heading as="h1" size="2x">
  Views
</Heading>
<Text size="sm" className="mt-2 text-muted-foreground">
  Multi-view table architecture. When a table is defined, it declares which views it supports. The UI provides a view
  toggle to switch between them and each view renders the same data through a different lens.
</Text>

<Heading as="h2" size="lg" id="view-types" data-heading>
  View Types
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  Seven view types are supported. Timeline is a config variant of Gantt. Each table defines which subset is available
  via <code>supportedViews</code>.
</Text>

| View | Description | Use case |
|------|-------------|----------|
| **Grid** | Spreadsheet-like rows and columns with inline editing | Default data entry, bulk editing |
| **List** | Vertical card or condensed row layout | Contacts, products, content items |
| **Kanban** | Column-based board grouped by a field | Status tracking, pipeline management |
| **Calendar** | Date-driven month/week/day rendering | Events, scheduling, deadlines |
| **Gantt + Timeline** | Timeline bars with dependencies and milestones; `mode="timeline"` strips dependencies for chronological event streams | Project planning, task scheduling, activity logs |
| **Workload** | Resource allocation and capacity visualization | Team planning, resource scheduling |
| **Form** | Individual row edit/insert as a full form | Detailed record editing, data entry |

<Heading as="h2" size="lg" id="architecture" data-heading>
  Architecture
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  The view system is built in three layers, consistent with the component architecture.
</Text>

<Heading as="h3" size="md" id="layer-1" data-heading>
  Layer 1: Primitives
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  Existing UI components like Table, Calendar, and Surface. Each view is composed from these primitives and inherits
  their <code>variant</code>, <code>color</code>, and <code>size</code> props.
</Text>

<Heading as="h3" size="md" id="layer-2" data-heading>
  Layer 2: View Components
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  Each view type is a standalone component such as <code>GridView</code>, <code>ListView</code>, or
  <code> KanbanView</code> that accepts table data and a view-specific config.
</Text>

<Heading as="h3" size="md" id="layer-3" data-heading>
  Layer 3: TableView Orchestrator
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  The top-level <code>TableView</code> component manages the view toggle, routes to the active view component, and
  resolves related table lookups.
</Text>

<Heading as="h2" size="lg" id="lookups" data-heading>
  Related Table Lookups
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  Tables reference other tables for foreign-key relationships. Lookups render differently depending on the active
  view.
</Text>

<LiveCodeBlock
  showPreview={false}
  initialCode={`interface LookupDefinition {
  field: string
  targetTable: string
  targetField: string
  displayField: string
  displayMode: 'inline' | 'select' | 'autocomplete' | 'dialog'
}`}
/>

| View | Lookup rendering |
|------|-----------------|
| **Grid** | Display value inline, click to expand related record |
| **Form** | Select / Autocomplete with search against related table |
| **List** | Linked chip or badge on card |
| **Kanban** | Chip on card, can serve as grouping column |
| **Calendar** | Metadata on event popover |
| **Gantt** | Metadata on bar tooltip |
| **Timeline** | Linked chip on event card |
| **Workload** | Resource name from lookup table |

<Heading as="h2" size="lg" id="form-view" data-heading>
  Form View
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  The Form view provides a full-form layout for individual record editing and insertion. It builds on the AutoForm
  architecture, so JSON Schema drives field rendering with type-appropriate components like TextField, DatePicker, and
  Select.
</Text>

| Mode | Behaviour |
|------|-----------|
| **Edit** | Load existing row, modify fields, save |
| **Insert** | Blank form with defaults, create new row |
| **Read-only** | View record details without editing |

<Heading as="h2" size="lg" id="gantt-implementation" data-heading>
  Gantt + Timeline Implementation
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  The Gantt view is built from scratch using React SVG components on the design system&apos;s token layer. Core
  algorithms like date-to-pixel positioning, snap-to-grid, and arrow path routing are adapted from
  <a href="https://github.com/frappe/gantt" target="_blank" rel="noopener noreferrer"> frappe-gantt</a> and
  transposed onto our token system.
</Text>

<Text size="sm" className="mt-2 text-muted-foreground">
  Timeline is a config variant of Gantt (<code>mode="timeline"</code>) that strips dependencies, progress bars, and
  milestones while sharing the same component and drag interactions.
</Text>

<Heading as="h2" size="lg" id="dark-mode" data-heading>
  Dark Mode
</Heading>

<Text size="sm" className="mt-2 text-muted-foreground">
  All views support dark mode automatically via semantic color tokens. No view should use hardcoded hex or rgb values.
  SVG elements and interactive states must remain visually distinct in both themes.
</Text>
