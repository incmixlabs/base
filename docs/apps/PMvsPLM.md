
  PM (Product Management) — manages the product strategy and roadmap. Market-centric: feature prioritization, user research,
  requirements, go-to-market. Users: product managers, stakeholders.

  ┌───────────┬──────────────────────────────────────┬───────────────────────────────┐
  │           │                 PLM                  │              PM               │
  ├───────────┼──────────────────────────────────────┼───────────────────────────────┤
  │ Focus     │ How it's built & manufactured        │ What to build & why           │
  ├───────────┼──────────────────────────────────────┼───────────────────────────────┤
  │ Artifacts │ BOMs, CAD, ECOs, specs               │ PRDs, roadmaps, user stories  │
  ├───────────┼──────────────────────────────────────┼───────────────────────────────┤
  │ Lifecycle │ Concept → Design → Manufacture → EOL │ Discovery → Delivery → Growth │
  ├───────────┼──────────────────────────────────────┼───────────────────────────────┤
  │ Tools     │ Siemens Teamcenter, PTC Windchill    │ Jira, Linear, Productboard    │
  ├───────────┼──────────────────────────────────────┼───────────────────────────────┤
  │ Data      │ Part numbers, revisions, suppliers   │ Features, sprints, OKRs       │
  └───────────┴──────────────────────────────────────┴───────────────────────────────┘

  They overlap at requirements — PM defines what the product should do, PLM tracks how it's realized in engineering. In integrated
  orgs, PM feeds requirements into PLM's design phase.

  For your PIM/PLM/PDM system, PM tooling (roadmaps, prioritization) would be a separate layer on top — think Linear/Jira territory,
  not the same data model.
